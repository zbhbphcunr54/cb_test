## 一、系统全貌

### 核心设计原则

- 交易信号精细分析：动态 Top 20 币种（按日均成交额排名，每日自动更新）
- 异动监控全市场覆盖：OKX 全部现货 + 合约币种（700+）
- 波浪图每小时更新：仅 Top 20 币种
- 所有信号必须过风控 + 盈亏比门槛，不满足条件不推送
- 期望值公式驱动：`期望值 = 胜率 × 平均盈利 - 败率 × 平均亏损 > 0`

## 三、数据采集层

### 3.1 动态 Top 20 机制

```
每日 00:05 UTC 执行：

1. 调用 OKX GET /api/v5/market/tickers 获取全市场 24H 成交额
2. 现货 + 合约去重（同一标的只算一次，取合约成交额）
3. 按成交额降序排名，取前 20
4. 与昨日 Top 20 对比：
   - 新进入 → 初始化精细数据采集 + 波浪计数
   - 掉出   → 停止精细采集 + 推送提醒
5. 推送「今日 Top 20 更新」消息
```

### 3.2 全市场轻量扫描

```
每 15 分钟执行：

GET /api/v5/market/tickers?instType=SPOT   → 全部现货（1 次调用）
GET /api/v5/market/tickers?instType=SWAP   → 全部合约（1 次调用）

仅 2 次 API 调用即可覆盖全市场，提取：
  - 最新价、24H 成交额、24H 涨跌幅
  - 合约额外：持仓量、资金费率

数据写入内存缓存 + SQLite（用于计算历史均值）
```

### 3.3 Top 20 精细数据

```
WebSocket 实时订阅（Top 20）：
  - K 线：1m / 5m / 15m / 1H / 4H / 1D
  - 深度：asks/bids Top 5
  - 成交：逐笔成交流（用于 Taker 买卖统计）

REST 补充（每小时）：
  - 历史 K 线（回填，确保数据完整）
  - 资金费率历史
  - 持仓量历史
```

### 3.4 API 限频管理

```
OKX REST 限频：20 次/2 秒（按 IP）

应对策略：
  - 令牌桶限流器，精确控制请求频率
  - 优先使用批量接口（tickers 一次返回全市场）
  - Top 20 精细数据优先用 WebSocket，减少 REST 调用
  - 预估消耗：全市场 tickers 2次/15min + Top20 ~200次/hour → 远低于限频
```

## 四、P1 中长线引擎

> 作用于 Top 20 币种，4H / 1D 级别
> 目标：吃大波段趋势，胜率 35%-45%，盈亏比 2.5:1 - 4:1

### 4.1 市场状态分类（前置过滤）

中长线引擎的第一步不是找信号，而是判断当前市场是否适合做趋势交易：

```
使用 ADX（Average Directional Index）+ 布林带宽度双重确认：

趋势市（策略正常运行）：
  ADX(14) > 25 且布林带宽度处于扩张状态

震荡市（策略暂停）：
  ADX(14) < 20 且布林带宽度持续收窄

过渡状态（降低仓位 50%）：
  ADX 20-25 之间
```

### 4.2 趋势方向确认

```
均线不用于产生交易信号，仅用于确认方向：

多头环境：日线 EMA20 > EMA60 且 EMA20 斜率向上
空头环境：日线 EMA20 < EMA60 且 EMA20 斜率向下
中性：    EMA20 与 EMA60 缠绕 → 不做

原则：只做顺势单
  多头环境 → 只找做多机会
  空头环境 → 只找做空机会
```

### 4.3 入场信号（回踩入场，非交叉入场）

```
不使用均线交叉作为入场信号（太滞后）
改用「趋势方向 + 回踩关键位 + 确认信号」三步入场：

第一步：确认多头/空头环境（4.2）

第二步：等待价格回踩到关键位
  关键位的定义（可量化，无主观判断）：
  · 4H EMA20（短期均值回归）
  · Fibonacci 回撤位（前一波段的 38.2% / 50% / 61.8%）
  · 前期 Swing High/Low（前高变支撑 / 前低变阻力）
  · 成交量密集区（Volume Profile POC 附近）

第三步：确认信号（价格在关键位附近企稳）
  · 连续 2 根 4H K 线收盘价回到关键位上方（做多）
  · 或 RSI 从超卖区（<35）回升至 >40（做多）
  · 或 MACD 柱状图连续缩短 3 根以上（空头动能衰竭）

三步全部满足 → 生成候选信号 → 进入盈亏比评估
```

### 4.4 盈亏比评估（硬性门槛）

```python
def evaluate_long_trade(entry, swing_low, prev_swing_high):
    """
    入场价、最近摆动低点（止损参考）、前一波段高点（目标参考）
    """
    stop_loss = swing_low * 0.998  # 摆动低点下方 0.2%
    take_profit_1 = prev_swing_high  # 目标一：前高
    take_profit_2 = entry + (entry - stop_loss) * 3  # 目标二：3R

    risk = entry - stop_loss
    reward = take_profit_1 - entry
    rr_ratio = reward / risk

    # 硬性门槛：盈亏比 < 2.5 直接丢弃
    if rr_ratio < 2.5:
        return None

    return signal
```

### 4.5 止盈策略（三段式）

```
目标一（2R）：平仓 30%，将止损移至入场价（保本）
目标二（3R）：平仓 30%
剩余 40%：  用 SuperTrend 指标做移动止损，跟踪趋势直到反转
```

### 4.6 止损策略

```
初始止损：取以下两者中更合理的一个
  · 结构止损：Swing Low 下方（做多）或 Swing High 上方（做空）
  · ATR 止损：入场价 ± ATR(14) × 2

止损一旦设定，只能收紧，不能放宽
```

### 4.7 额外过滤条件

```
· 重大事件前 24 小时（CPI、FOMC、非农）不开新仓，已有仓位收紧止损
· 全网合约持仓量处于历史 95% 分位以上时，仓位减半
· 资金费率绝对值 > 0.1% 时，不做同方向的单（费率过高 = 一侧拥挤）
```

## 五、P2 异动监控引擎

> 作用于全市场 700+ 币种
> 目标：发现异常，不预测方向，作为预警系统

### 5.1 检测维度

| 维度 | 数据来源 | 检测方法 | 触发条件 |
|------|---------|---------|---------|
| 成交量突变 | tickers 批量接口 | Z-Score（回看 50 个周期） | Z > 3.0 |
| 价格异动 | tickers 批量接口 | 15 分钟涨跌幅 | > ±5% |
| 多币联动 | 以上数据聚合 | 同时异动币种计数 | ≥ 5 个币同时放量 |
| OI 异动 | 合约 tickers | OI 变化率 vs 价格变化率 | OI 涨 >10% 且价格波动 <2% |
| 资金费率极值 | 定时拉取 funding rate | 绝对值排名 | > 0.1% 或 < -0.1% |

### 5.2 确认窗口机制（降低误报）

```
异动检测不是"触发即推送"，而是加一个确认步骤：

第一次检测到异动 → 标记为「待确认」，不推送
15 分钟后复查：
  · 异动持续或加剧 → 确认，推送
  · 已回归正常 → 丢弃，视为噪音

例外：多维度同时异动（量 + 价 + OI 至少两项）→ 跳过确认，直接推送
```

### 5.3 异动分级

```
🔴 一级异动（立即推送）
   · 多维度同时触发（量 + 价 + OI 至少两项异常）
   · 或单币种 15 分钟涨跌幅 > ±10%
   · 或多币联动（≥5 个币种同时异动）

🟡 二级异动（每 15 分钟汇总为一条消息推送）
   · 单维度异常且通过确认窗口
   · 或资金费率极值
```

### 5.4 与交易引擎联动

```
异动引擎检测到某币种异动后，通知短线/中长线引擎：

· 多头异动（放量 + Taker 买入主导 + 价格在支撑位上方）
  → 短线引擎该币种做多信号优先级提升
  → 中长线引擎若有该币种持仓，持仓信心增强

· 空头异动（放量 + Taker 卖出主导 + OI 暴增）
  → 该币种已有多单 → 推送收紧止损建议
  → 短线引擎暂停该币种做多信号

· 全市场联动异动
  → 可能是系统性事件（黑天鹅/监管消息/交易所故障）
  → 所有引擎进入防御模式，不开新仓
```

### 5.5 成交量 Z-Score 算法

```python
import numpy as np

def detect_volume_anomaly(volumes, lookback=50, threshold=3.0):
    """
    基于 Z-Score 的成交量异动检测
    volumes: 历史成交量序列
    lookback: 回看窗口
    threshold: Z-Score 阈值（3.0 约等于 top 0.13%）
    """
    if len(volumes) < lookback:
        return None

    recent = volumes[-lookback:-1]  # 排除当前周期
    current = volumes[-1]

    mean = np.mean(recent)
    std = np.std(recent)

    if std == 0:
        return None

    z_score = (current - mean) / std

    if z_score > threshold:
        return {
            'type': 'VOLUME_SPIKE',
            'z_score': round(z_score, 2),
            'current_vol': current,
            'avg_vol': round(mean, 2),
            'multiple': round(current / mean, 1)
        }
    return None
```

### 5.6 OI 异动算法

```python
def detect_oi_anomaly(oi_series, price_series, lookback=24):
    """
    持仓量暴增 + 价格平稳 = 变盘前兆
    """
    oi_change = (oi_series[-1] - oi_series[-lookback]) / oi_series[-lookback]
    price_change = abs(price_series[-1] - price_series[-lookback]) / price_series[-lookback]

    if oi_change > 0.10 and price_change < 0.02:
        return {
            'type': 'OI_DIVERGENCE',
            'oi_change': f"+{oi_change:.1%}",
            'price_change': f"{price_change:.1%}",
            'warning': '持仓量大幅增加但价格未动，注意变盘'
        }
    return None
```

## 六、P3 短线引擎

> 作用于 Top 20 币种，5m / 15m 级别
> 目标：捕捉日内至 1-3 天的快速波段，胜率 >50%，盈亏比 1.5:1 - 2:1
> 核心约束：必须有异动引擎配合才推送信号

### 6.1 设计原则

```
短线引擎在币圈是最难稳定盈利的部分
因此采用保守设计：

1. 不单独推送 — 必须与异动引擎共振
2. 规则极简 — 越简单越不容易过拟合
3. 频率严控 — 宁可错过，不可乱做
```

### 6.2 趋势过滤

```
1H EMA20 方向作为大方向过滤：
  EMA20 向上 → 只做多
  EMA20 向下 → 只做空
  EMA20 走平 → 不做
```

### 6.3 入场信号（极简规则）

```
不使用复杂 K 线形态识别（吞没/锤子线等自动识别误判率高）
改用简单可量化的规则：

做多条件（全部满足）：
  ① 1H EMA20 向上（趋势方向）
  ② 15m 价格回踩至 EMA9 或 VWAP 附近（±0.3%）
  ③ 连续 2 根 15m K 线收盘价走高（企稳确认）
  ④ 当根成交量 > 前 20 根均量 × 1.5（量能确认）
  ⑤ 异动引擎对该币种无空头异动标记

做空条件：以上条件镜像反转
```

### 6.4 异动共振要求（关键改进）

```
纯技术面短线信号 → 不推送（误判率太高）
技术面 + 异动引擎同时触发 → 才推送

共振场景举例：
  · 15m 出现做多信号 + 5m 成交量 Z-Score > 2.5 + Taker 买入占比 > 60%
  → 推送

  · 15m 出现做多信号，但成交量正常，Taker 无明显偏向
  → 不推送，记录日志但不打扰
```

### 6.5 盈亏比门槛

```python
# 短线盈亏比门槛：R:R >= 1.5
stop_loss = 反转 K 线极值外侧 或 EMA9 反向 0.5%
take_profit_1 = 1.5 × 止损距离（平仓 50%）
take_profit_2 = 移动止损跟踪剩余 50%

if risk_reward < 1.5:
    discard()  # 丢弃
```

### 6.6 止盈策略（两段式）

```
目标一（1.5R）：平仓 50%，止损移至入场价
剩余 50%：    15m EMA9 作为移动止损线，跌破平仓
```

### 6.7 超时强平

```
持仓超过 72 小时未达目标 → 强制平仓
短线就是短线，不允许变成"被套的中长线"
```

### 6.8 频率限制

```
· 同一币种 24H 内最多 2 笔短线单
· 全市场每小时最多推送 5 条短线信号
· 资金费率绝对值 > 0.1% 时不开新仓
· 大 K 线收线前 5 分钟不开仓（避免收线假突破）
```

## 七、P4 波浪分析引擎

> 作用于 Top 20 币种，每小时更新
> 定位：辅助工具，提供结构背景，不直接产生交易信号

### 7.1 设计原则

```
波浪自动计数的准确率约 50%-60%，这是波浪理论本身的特性决定的
因此本模块采用以下策略：

1. 降低颗粒度 — 不追求精确标注每一子浪，重点判断"阶段"
2. 给出多方案 — Top 3 计数方案 + 置信度评分
3. 交叉验证 — 波浪结论与传统指标互相校验
4. 定期复盘 — 回顾历史计数，校准算法参数
```

### 7.2 阶段判断（核心输出）

```
不追求精确的 ①②③④⑤ 标号
优先输出三种状态（准确率可达 70%+）：

· 推动阶段 — 趋势主方向运行中，顺势操作
· 调整阶段 — 逆趋势回调中，等待调整结束
· 模糊状态 — 无法判断，标记为"观望"
```

### 7.3 算法流程

**第一步：ZigZag 提取转折点**

```
多级别 ZigZag 参数：
  短线（对应子浪）：阈值 1%-2%
  中线（对应主浪）：阈值 3%-5%
  长线（对应大浪）：阈值 8%-15%

输出：Swing High / Swing Low 序列
```

**第二步：波浪规则校验**

```
三条不可违反的铁律：
  ① 2 浪不能回撤超过 1 浪起点
  ② 3 浪不能是 1/3/5 浪中最短的
  ③ 4 浪不能进入 1 浪的价格区域

核心指引（非强制，用于评分）：
  · 2 浪常回撤 1 浪的 50%-78.6%
  · 3 浪常是 1 浪的 1.618 / 2.618 延伸
  · 4 浪常回撤 3 浪的 23.6%-38.2%
  · 5 浪常等于 1 浪长度，或为 1 浪的 0.618 / 1.618
  · 2 浪和 4 浪交替（一个简单一个复杂）
```

**第三步：模式匹配 + 评分**

```
对每一组可能的波浪计数打分：

得分维度：
  · Fibonacci 比例吻合度     权重 30%
  · 时间对称性              权重 15%
  · 成交量配合              权重 20%
    （3 浪量最大，5 浪量背离）
  · 规则满足度              权重 25%
    （三大铁律是否全部通过）
  · 子浪结构完整性           权重 10%

输出：按分数排序的 Top 3 波浪计数方案
```

**第四步：与传统指标交叉验证**

```
波浪结论 + 指标信号互相校验，调整置信度：

一致 → 置信度提升
  波浪说"③浪进行中" + MACD 持续放大 + 成交量递增 → 高置信

矛盾 → 置信度降级
  波浪说"③浪" 但 MACD 背离 + 缩量 → 可能计数错误，降级处理
```

### 7.4 多时间框架嵌套

```
每个币种同时分析三个级别：

日线波浪（大周期方向）
  └─ 4H 波浪（中周期结构）
      └─ 1H 波浪（短周期位置）

嵌套判读示例：
  日线③ + 4H ⑤ + 1H ④

```

### 8.1 仓位管理

```python
def calc_position_size(balance, risk_pct, entry, stop_loss):
    """
    固定风险比例仓位计算
    每笔交易风险固定为总资金的 N%
    不管止损宽窄，亏损金额恒定
    """
    risk_per_trade = balance * risk_pct
    stop_distance = abs(entry - stop_loss)
    position_size = risk_per_trade / stop_distance
    return position_size
```

| 引擎 | 单笔风险 | 建议杠杆 |
|------|---------|---------|
| 短线 | ≤ 1% 总资金 | 5x-10x |
| 中长线 | ≤ 2% 总资金 | 3x-5x |

### 8.2 风控规则

| 规则 | 说明 |
|------|------|
| 单笔风险上限 | 短线 ≤1%，中长线 ≤2% |
| 总敞口上限 | 所有持仓合计风险 ≤ 总资金 10% |
| 同币种限制 | 不允许同一币种同时存在方向相反的短线和中长线单 |
| 频率限制 | 单币种 24H 内最多 2 笔短线 + 1 笔中长线 |
| 回撤熔断 | 周回撤 >8% → 仓位减半；周回撤 >15% → 暂停一周 |
| 期望值监控 | 滚动 30 笔期望值转负 → 暂停该引擎并告警 |
| 盈亏比硬性门槛 | 短线 R:R < 1.5 丢弃，中长线 R:R < 2.5 丢弃 |
| 市场状态过滤 | ADX < 20 震荡市 → 暂停中长线引擎 |
| 全市场联动异动 | 触发 → 所有引擎进入防御模式，不开新仓 |

### 8.3 绩效统计

```python
def update_stats(trades):
    """
    实时计算策略表现
    """
    wins = [t for t in trades if t['pnl'] > 0]
    losses = [t for t in trades if t['pnl'] <= 0]

    win_rate = len(wins) / len(trades) if trades else 0
    avg_win = sum(t['pnl'] for t in wins) / len(wins) if wins else 0
    avg_loss = abs(sum(t['pnl'] for t in losses) / len(losses)) if losses else 1

    actual_rr = avg_win / avg_loss
    expectancy = win_rate * avg_win - (1 - win_rate) * avg_loss

    return {
        'total_trades': len(trades),
        'win_rate': win_rate,
        'avg_rr': actual_rr,
        'expectancy': expectancy,
        'max_drawdown': calc_max_drawdown(trades)
    }
```

```python
    """推送 markdown 文字消息到指定群"""
    payload = {
        "msgtype": "markdown",
        "markdown": {"content": content}
    }
    requests.post(WEBHOOKS[group], json=payload)

def send_image(group: str, image_path: str):
    """推送图片到指定群（波浪图）"""
    with open(image_path, 'rb') as f:
        image_data = f.read()

    b64 = base64.b64encode(image_data).decode()
    md5 = hashlib.md5(image_data).hexdigest()

    payload = {
        "msgtype": "image",
        "image": {"base64": b64, "md5": md5}
    }
    requests.post(WEBHOOKS[group], json=payload)

def send_to_all(content: str):
    """推送到所有群（风控告警等紧急消息）"""
    for group in WEBHOOKS:
        send_text(group, content)

def send_with_cc(primary_group: str, cc_groups: list, content: str):
    """推送到主群 + 抄送到其他群"""
    send_text(primary_group, content)
    for group in cc_groups:
        send_text(group, f"[抄送自{primary_group}]\n\n{content}")
```

## 十三、开发阶段

建议按阶段上线，每个阶段交付一个可用版本，不要等全部做完再上线。

## 十四、各模块靠谱程度评估

| 模块 | 准确度 | 滞后性 | 实战价值 | 改进要点 |
|------|--------|--------|---------|---------|
| **P2 异动监控** | 高 | 极低（实时） | 最高 | 确认窗口降低误报 |
| **P1 中长线** | 中等 | 中等（改进后可接受） | 高 | 回踩入场替代交叉入场 + 市场状态分类 |
| **P3 短线** | 中低 | 低 | 中等 | 必须与异动共振才推送 |
| **P4 波浪** | 低-中 | 不适用 | 辅助参考 | 降低颗粒度，判断阶段而非精确标号 |

**核心认知：系统的盈利能力不取决于任何单一模块的准确度，而取决于风控是否严格执行。胜率 40% 的策略只要盈亏比 3:1 + 仓位管理到位，长期就是正期望值。**
